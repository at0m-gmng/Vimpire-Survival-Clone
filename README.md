# Vampire Survival Clone

Клон популярного жанра Vampire Survivors, созданный на Unity 6000.0 с использованием современных подходов к архитектуре и dependency injection.

## Описание проекта
1) Управление персонажем WASD через NewInputSystem. Добавить стик на Canvas и будет мобильное работать.
2) Авто-атака (выглядит как появляющаяся вокруг игрока сфера)
3) Авто-подбор опыта в радиусе.
4) Система уровней с постепенной прогрессией
5) 3 реализованных умения. 
5.1) Снаряд (зелёный шар). При повышении уровня больше шаров и лучше характеристик.
5.2) Аура (фиолетовая сфера вокруг). Умение Ауры стакается при повторном выборе.
5.3) Орбита (оранжевые шары вокруг). При повышении уровня увеличение снарядов, урон, откат.
6) 2 типа врагов. Красные быстрые. Жёлтые медленные.
7) Система выбора способностей

### Требования
- Unity 6000.0 или выше
- Установленные пакеты (автоматически загружаются из Package Manager):
  - Input System 1.14.0
  - Addressables 2.5.0
  - UniTask (из GitHub)
  - Zenject (встроен в проект)
  - UniRx (встроен в проект)

### Запуск проекта Unity

1. Откройте проект в Unity 6000.0
2. Дождитесь импорта всех зависимостей
3. Откройте сцену `/Assets/GameResources/Scenes/SampleScene.unity`
4. Нажмите Play

### Запуск проекта WebGL

1. Откройте index.hxml в Rider
2. В правом верхнем углу появятся иконки для локального хоста
3. Выберете Rider или GoogleChrome
4. Играйте

### Управление
Управление настраивается через новую Input System и конфигурируется в `InputConfig` ScriptableObject.

## Архитектура проекта

Проект использует модульную архитектуру с разделением ответственности и dependency injection через Zenject.

### Основные системы

#### **Entry Point System**
Точка входа в игру, инициализирует все системы, загружает конфигурацию из JSON и запускает системы спавна в правильном порядке.

#### **Dependency Injection (Zenject)**
Главный инсталлер DI контейнера регистрирует все фабрики, системы и конфигурации. Настраивает object pooling для оптимизации производительности.

#### **Ability System**
Система способностей игрока с различными типами атак: снаряды, аура урона, триггерный урон и дальние атаки.

#### **Spawn System**
Система спавна сущностей с иерархией базовых классов для игрока, врагов и коллекционных предметов с автоматической генерацией позиций.

#### **Factory Pattern**
Фабрики для создания игровых объектов с object pooling. Включает фабрики для игрока, врагов, снарядов и коллекционных предметов. Factory Managers управляют множественными типами через идентификаторы.

#### **Experience & Level Up System**
Система накопления опыта и повышения уровня с выбором наград.

#### **Health System**
Система здоровья для всех сущностей с поддержкой урона и смерти.

#### **Input System**
Использует новую Input System Unity для обработки ввода с поддержкой клавиатуры и геймпада.

#### **Camera System**
Система камеры, следующая за игроком с настройкой через `CameraConfig`.

#### **Movement System**
Система движения персонажей с физикой.

#### **Save/Load System**
Сохранение и загрузка данных в JSON, включая конфигурации игры.

#### **Pause System**
Система паузы игры.

#### **Signal Bus (Event System)**
Система событий на основе Zenject SignalBus для коммуникации между системами (убийство сущности, создание/уничтожение игрока, сбор опыта, повышение уровня и т.д.).

#### **UI System**
Views для отображения интерфейса игры.

#### **Data & Configs**
ScriptableObjects для конфигурации в `/Assets/GameResources/ScriptableObjects/Configs`, Prefab configs для различных типов сущностей и JSON конфигурации в `StreamingAssets/Configs`.

### Структура проекта

```
Assets/GameResources/
├── Materials/          # Материалы для визуальных эффектов
├── Prefabs/           # Префабы игровых объектов
│   ├── Collectables/
│   ├── Enemies/
│   ├── Players/
│   ├── Projectiles/
│   └── UI/
├── Scenes/            # Игровые сцены
├── ScriptableObjects/ # Конфигурации
└── Scripts/           # Исходный код
    ├── AbilitySystem/
    ├── CameraSystem/
    ├── ExperienceSystem/
    ├── Factories/
    ├── HealthSystem/
    ├── InputSystem/
    ├── LevelUpSystem/
    ├── MovementSystem/
    ├── SaveLoadSystem/
    ├── SpawnSystem/
    └── UI/
```

### Используемые паттерны
- **Dependency Injection** (Zenject)
- **Factory Pattern** с Object Pooling
- **Facade Pattern** для игровых сущностей
- **Signal/Event Bus** для коммуникации между системами
- **ScriptableObject** для data-driven конфигурации
- **Async/Await** с UniTask для асинхронных операций

## Использование AI

Проект активно использует AI для разработки.

### AI-ассистированная разработка
- **Прототип проекта**: - Использовался базовая версия Cursor для написания базовых примитивных механик.
- **Архитектурные решения**: - После исчерпания лимита Cursor использовал браузерные версии DeepSeek и немного Qwen для правки собственных архитектурных решений. После выстаивания логики дублировал классы с помощью ИИ.  
- **Генерация кода**: - После выстроения основной связки игры перешёл к написанию способностей полностью с помощью Bezi AI.
- **Рефакторинг**: - Также с помощь Bezi AI поправил некоторые решения прошлых итерацию, придумав более удобные архитектурные моменты.
- **Документация**: - Автоматическая генерация документации с помощью Bezi AI
